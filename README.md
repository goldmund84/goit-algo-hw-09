### Завдання 1 — решта монетами

- Функція `find_coins_greedy(amount)` будує решту, обираючи найбільші доступні номінали. Часова складність O(k) для k номіналів: навіть при великих сумах алгоритм швидкий, але може давати неоптимальні рішення, якщо монети не канонічні.
- Функція `find_min_coins(amount)` реалізує динамічне програмування (аналізує всі суми до `amount`), тому працює за O(amount × k), зате гарантує мінімальну кількість монет.
- У поточному наборі [50, 25, 10, 5, 2, 1] обидва підходи дають однаковий результат (наприклад, для 113: {50: 2, 10: 1, 2: 1, 1: 1}), оскільки номінали утворюють канонічну систему. Для великих сум жадібний алгоритм зростає лінійно, тоді як DP — лінійно від суми; однак DP є універсальним і залишається правильним, якщо змінити номінали.
- Практичне рішення: застосовувати жадібний алгоритм для стандартного набору монет через високу швидкість, але реалізовувати резервну DP-функцію (наприклад, для тестів або нестандартних валют).
